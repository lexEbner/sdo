Ontology Specification Draft

Abstract

Industrial systems generate massive streams of time series data, but differences in formats, technologies, and timing make this data hard to interpret and share. The ontology presented here provides a common language to describe such data in the form of signals. At its core, it offers a technology-neutral model that captures the essential characteristics of signals across diverse sources. On top of this, it supports specialized extension modules that adapt the model to specific technologies and provide all the details needed for actual data access. This approach makes signals easier to discover, understand, and use.

Introduction back to ToC

Modern industrial systems generate enormous amounts of data. Machines, sensors, and software exchange information at high speed, supporting everything from real-time process control to supply chain optimization. But with this growth comes a major challenge: heterogeneity. Different devices and platforms use their own formats, semantics, and access methods. The result is a patchwork of isolated data silos that are hard to integrate and even harder to interpret. To unlock the full potential of this data, we need a shared way of describing and accessing it. Semantic Web technologies already offer powerful tools for creating machine-interpretable interfaces across diverse sources, enabling true interoperability. However, when it comes to time series data, additional challenges emerge. Differences in timestamps, irregular sampling, and jitter make consistent analysis difficult. The concept of signals helps address these issues by treating time series as continuous functions of time. Signals make it possible to work with data in a consistent way, regardless of where it originates or how it is sampled. What's been missing, however, is a way to combine this concept with semantic technologies to also overcome the data access barriers. This ontology fills that gap. It provides a technology-agnostic foundation for describing signals, along with a flexible framework for building extensions that connect to specific technologies. With this ontology, signals can be described in a uniform way and linked to the mechanisms needed to access them. This makes heterogeneous time series data easier to discover, integrate, and use across different technologies and domains.
